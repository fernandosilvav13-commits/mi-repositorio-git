# Plan 13-01: Two-Pass Pipeline Orchestrator

**Phase:** 13 - Two-Pass Pipeline
**Plan:** 01
**Wave:** 1
**Objective:** Create ExtractionPipeline orchestrator that chains preprocessing → classification → extraction with type-specific prompts.
**Gap closure:** false

## Tasks

### Task 1: Create ExtractionPipeline

File: `backend/app/services/extraction_pipeline.py`

Pipeline class with module-level singleton:
- `process(raw_text: str) -> dict`: main entry point
  - Step 1: Preprocess via `preprocessing_pipeline.process(text)`
  - Step 2: Classify via `doc_classifier.classify(cleaned_text)`
  - Step 3: If CV → resolve type-specific prompt via `prompt_resolver.get("cv-extraction", "^v1.0.0")`
  - Step 4: Extract via `extract_fields()` with the resolved prompt
  - Step 5: Return result dict with extraction data + classification info + prompt version

### Task 2: Handle non-CV classification

If classification is "non-cv" or confidence < threshold:
- Return dict with `classification_warning: true`, `category: "non-cv"`, no extraction attempt
- Log the decision

### Task 3: LlmService prompt override

Modify `llm_service.extract_fields()` OR create a wrapper that accepts a prompt string instead of using EXTRACTION_PROMPT constant. The cleanest approach: add `prompt_override` parameter to `extract_fields()`.

## Verification

- Pipeline creates correct prompt for CV documents
- Non-CV documents return classification_warning
- Pipeline returns structured result
