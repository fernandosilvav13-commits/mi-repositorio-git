<objective>
Implement the three standalone pipeline stages: SectionDetector, NoiseFilter, LayoutNormalizer.

Purpose: Build the core preprocessing logic per D-10 (multi-stage pipeline with separate classes). SectionDetector makes a single batched LLM call (D-05, D-06) returning section boundaries and noise regions. NoiseFilter strips detected noise lines (D-07). LayoutNormalizer applies intelligent structure: paragraph breaks preserved, section boundary markers added (D-08), bullet points normalized (D-09).

    **SectionDetector:**
    - __init__(self, prompts_dir: str = "backend/prompts") — instantiate PromptResolver + _client = None
    - _get_client(self) -> genai.Client — lazy-init with settings.google_api_key (same pattern as llm_service.py)
    - async detect(self, text: str) -> DetectionResult:
        1. Load prompt via self.resolver.get("section-detection", "^v1.0.0")
        2. If prompt is None → log warning + return DetectionResult(sections=[], noise_regions=[]) (D-03 fallback)
        3. Render prompt via self.resolver.render(prompt_version, document_text=text)
        4. Call LLM: client.models.generate_content(model=..., contents=..., config={"response_mime_type": "application/json", "temperature": 0.1})
        5. Parse response.text with json.loads() → dict
        6. Return DetectionResult(**parsed)
        7. On JSON parse error: log error + return DetectionResult(sections=[], noise_regions=[]) (graceful degradation per D-03)
    - _repair_json(self, raw: str) -> str — mirror the repair logic from llm_service.py (strip markdown fences, fix trailing commas, etc.)

    Export SectionDetector in `backend/app/services/__init__.py`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM response → SectionDetector | Untrusted Gemini JSON response parsed into DetectionResult |
| Text input → NoiseFilter | Raw document text may contain embedded control characters |
| PromptResolver prompt → SectionDetector | Prompt YAML content controls LLM system instruction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-06 | Tampering | LLM returns malformed JSON | mitigate | _repair_json() handles common LLM issues; json.loads() wrapped in try/except; empty DetectionResult fallback |
| T-10-07 | Tampering | LLM returns negative line numbers | mitigate | Pydantic ge=0 validation on DetectionResult fields catches this during construction |
| T-10-08 | Denial of Service | LLM returns huge noise regions deleting all text | mitigate | NoiseFilter only removes lines that actually exist in the document — out-of-bounds indices are no-ops |
| T-10-09 | Tampering | LayoutNormalizer bullet regex matches non-bullet content | mitigate | Regex anchored to start of line with optional leading whitespace: `^[ \t]*[*•→▪▸-]\s+` |
| T-10-10 | Tampering | Section boundary markers collide with real text | accept | Markers use `[SECTION: ...]` format with uppercase — extremely unlikely in natural CV text |
| T-10-11 | Denial of Service | Extremely long document (100k+ lines) | accept | Processing is O(n) per stage. If needed, add line cap in future iteration. |
| T-10-12 | Elevation of Privilege | Prompt injection via section detection prompt | mitigate | Prompt YAML is dev-authored and committed to Git — not modifiable at runtime |
</threat_model>
