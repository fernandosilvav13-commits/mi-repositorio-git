    **PreprocessingPipeline:**
    - __init__(self, config: PreprocessingConfig | None = None):
        self.config = config or PreprocessingConfig()
        self.section_detector = SectionDetector()
        self.noise_filter = NoiseFilter()
        self.layout_normalizer = LayoutNormalizer(self.config)

    - async run(self, raw_text: str) -> str:
        1. Log pipeline start with text length
        2. Call self.section_detector.detect(raw_text) → DetectionResult
        3. If sections AND noise_regions are both empty: log "No sections/noise detected, passing raw text" + return raw_text (D-03)
        4. If noise_regions not empty: call self.noise_filter.filter(raw_text, result.noise_regions) → filtered_text
        5. If sections not empty: call self.layout_normalizer.normalize(filtered_text, result.sections) → normalized_text
        6. Log pipeline result stats (sections count, noise regions count, output length)
        7. Return normalized_text (or filtered_text if no sections, or raw_text if neither)
        8. Wrap whole flow in try/except: on any unexpected error, log error + return raw_text (fail-open)

    - run_batch(self, texts: list[str]) -> list[str]: Async gather of run() for multiple texts

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SectionDetector → Pipeline | DetectionResult flows from LLM into pipeline orchestration |
| Pipeline → cv_extractor.py | Normalized text flows into extraction — any corruption affects extraction quality |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-13 | Denial of Service | SectionDetector.detect() hangs or times out | mitigate | run() wraps in try/except — any exception returns raw_text (fail-open) |
| T-10-14 | Tampering | Pipeline drops content through chained stage errors | mitigate | Each stage is independent; if noise filter errors, pipe filtered text; if normalizer errors, pipe text before normalizer |
| T-10-15 | Information Disclosure | Section markers leak document structure in logs | accept | Logs contain section names which are standard CV section titles — no PII in section names |
| T-10-16 | Tampering | D-03 empty detection silently bypasses preprocessing | accept | Intentional design — if LLM cannot identify sections, raw text passes through to extraction unchanged |
| T-10-17 | Denial of Service | run_batch with 1000 texts causes resource exhaustion | mitigate | Current design uses asyncio.gather with no concurrency cap. Cap at 10 concurrent if needed (deferred to agent discretion). |
</threat_model>
