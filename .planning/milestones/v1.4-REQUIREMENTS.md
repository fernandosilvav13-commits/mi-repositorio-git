# Requirements Archive: v1.4 Extracción Inteligente

**Archived:** 2026-06-16
**Status:** ✅ SHIPPED

## Requirements

### Prompt Infrastructure

- [x] **PROMPT-01**: System supports version-controlled prompt registry with YAML files, resolver, and Git tracking — **Complete** (Phase 9)

### Document Classification

- [x] **CLASS-01**: System classifies documents using TF-IDF + SVM (scikit-learn) to detect document type before extraction — **Complete** (Phase 11, Phase 15)

### Post-Processing Rules

- [x] **RULES-01**: System expands field coverage with pattern-based inference rules beyond gender/phone/RUT — **Complete** (Phase 12)
- [x] **RULES-02**: New post-processing rules deploy in shadow mode with 90% precision floor before activation — **Complete** (Phase 12)

### Advanced Preprocessing

- [x] **PREP-01**: System applies structural cleanup (section detection, layout normalization) before extraction — **Complete** (Phase 10)
- [x] **PREP-02**: System filters noise (page headers, page numbers, artifacts) before extraction — **Complete** (Phase 10)

### OCR Augmentation

- [ ] **OCR-01**: System integrates PaddleOCR 3.0 as alternative OCR backend for image-based documents — **Deferred to v1.5** (Tesseract implemented, PaddleOCR not yet)
- [ ] **OCR-02**: System fuses Tesseract + PaddleOCR results with per-line confidence scoring — **Deferred to v1.5** (not implemented)
- [ ] **OCR-03**: System applies layout analysis (PP-StructureV3) for reading order recovery — **Deferred to v1.5** (coordinate-based layout implemented, not PP-StructureV3)

### Two-Pass Pipeline

- [x] **PIPE-01**: System wires classifier output into extraction with type-specific prompts — **Complete** (Phase 13, Phase 15)
