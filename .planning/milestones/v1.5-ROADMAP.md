## Milestone Summary

**Key Decisions:**

- Multi-provider LLM abstraction using factory pattern — enables provider flexibility without code changes
- Shadow rules evaluated for precision: 4 promoted, ANIOS_EXPERIENCIA kept shadow (not reliable across CV formats)
- OLE2 first, HTML fallback second for .doc parsing — prevents 6.5M character noise
- Auth dev bypass in dev mode: mock user when no token — avoids requiring Supabase for local development

**Issues Resolved:**

- 4 critical/major bugs fixed from Phase 19 real-CV validation
- 3 pre-existing test failures fixed (excel styling, ref prefixing, compound match keys)
- 269 tests passing (all green)

**Issues Deferred:**

- BUG-005: DOCX no fallback cascade (minor)
- BUG-006: antiword/catdoc not in FileParser (minor)

**Technical Debt Incurred:**

- batch_process.py still has hardcoded BATCH_EXTRACTION_PROMPT (should use PromptResolver in future)
- No E2E test suite for auth flow (requires Supabase configured)
- Phase directories (16, 20) have partial artifact coverage (executed inline)
