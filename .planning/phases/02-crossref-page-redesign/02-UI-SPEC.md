### Responsive Behavior

| Breakpoint | Upload Tile | File List Table |
|------------|-------------|-----------------|
| ≥ 1068px (desktop) | Full-width drop zone, centered | Full table, all columns visible |
| 834-1068px (tablet landscape) | Drop zone at 80% width | Table scrolls horizontally if needed |
| 640-833px (tablet portrait) | Drop zone full-width, compact padding | Table rows maintain info density |
| < 640px (phone) | Padding tightens (py-16), stack per-file progress vertically | Table shows only Name + Status; Actions in overflow menu or icon-only column |

**Touch targets:** All interactive elements ≥ 44×44px (upload button, delete icon buttons, drop zone click area).

## Spacing Scale

Using Apple spacing system from DESIGN.md — Tailwind v4 tokens map as follows:

| Token | Value | Usage |
|-------|-------|-------|
| xxs | 4px | File type badge gaps, icon-to-text gaps |
| xs | 8px | Badge gap spacing, row internal padding |
| sm | 12px | Compact element spacing, PillChip internal padding |
| md | 17px | Table cell vertical padding, Drop zone text-to-border gap |
| lg | 24px | Card internal padding, between drop zone and format badges |
| xl | 32px | Section heading vertical gaps, table margin from container edge |
| xxl | 48px | Between upload progress bars, between end of upload and list |
| section | 80px | Tile vertical padding (py-20 → 80px) — matches Tile.tsx |

**Exceptions:**
- Drop zone internal vertical padding: 64px (py-16) — not a standard token but matches Apple's generous whitespace aesthetic
- Table row minimum height: 52px — matches Apple sub-nav height, gives enough touch target for mobile
- Floating back button: position at top-left with 32px offset from edges

## Interaction States

### Drop Zone

| State | Visual |
|-------|--------|
| Default | Dashed border `rgba(0,0,0,0.08)`, centered content, rounded-lg (18px) |
| Hover | Border → `#0066cc` (action-blue), background → `rgba(0,102,204,0.03)` |
| Drag over | Border → `#0066cc` solid 2px, background → `rgba(0,102,204,0.08)`, subtle scale transform |
| Active (click) | `active-scale` transform (scale 0.95) |
| Focus-visible | `outline: 2px solid #0071e3` (focus-blue ring) |

### Upload Progress (Per File)

| Phase | Visual |
|-------|--------|
| Queued | File name + thin indeterminate blue bar (animate-pulse) |
| Uploading | File name + progress bar (blue fill, width based on XHR `upload.onprogress`) |
| Uploading label | "{filename} — {percent}%" |
| Complete | File name + green checkmark + "Complete" label |
| Failed | File name + red X + error message from server "Upload failed — {reason}" |

Progress bar: Apple-style thin (4px height) rounded bar. Background `rgba(0,0,0,0.08)`, fill `#0066cc` (or `#34c759` on complete). No text inside bar — percentage shown as label next to bar.

### Frosted Table Rows

| State | Visual |
|-------|--------|
| Default | `bg-white/40 backdrop-blur-md border-b border-white/20` |
| Hover | `bg-white/60` with 300ms transition |
| Active (click row) | Not applicable (rows are not clickable) |

### Upload-to-List Animation

When upload completes:
1. New file entry appears at the top of the table
2. Animates in with `animate-in fade-in slide-in-from-top-2 duration-500 fill-mode-both`
3. If user uploaded multiple files, each new row staggers by 100ms delay

### Delete Action

1. Click trash icon → Dialog opens
2. Dialog: "Delete {filename}?" with Cancel (ghost) and Delete (destructive red) buttons
3. On confirm: file removed from list with `animate-out fade-out duration-300`
4. On cancel: dialog closes, no action

### Button Active State (System-Wide)

All interactive elements use the Apple `active-scale` utility: `transform: scale(0.95)` on `:active`. Defined in `globals.css` and used across all buttons, chips, and clickable containers.

## Accessibility & Interaction Requirements

| Requirement | Specification |
|-------------|---------------|
| Keyboard navigation | All interactive elements focusable and activatable via keyboard (Tab, Enter, Space) |
| Focus indicators | `focus-visible:outline-2 focus-visible:outline-[#0071e3]` on all interactive elements |
| Screen reader | Drop zone: `role="button"` with `aria-label="Upload cross-reference files"`. Table: standard `<table>` markup via shadcn. Status pills: `aria-label="Status: matched"`. Delete icons: `aria-label="Delete {filename}"` |
| Upload progress | `aria-valuenow`, `aria-valuemin`, `aria-valuemax` on progress bars |
| Drag-and-drop | Native DnD API (`onDragOver`, `onDrop`) with `preventDefault()`. Include hidden file input as fallback for keyboard users |
| Reduced motion | `prefers-reduced-motion` honored: animations degrade gracefully to instant transitions |
| Color contrast | Status badges use `bg-{color}/10` backgrounds with solid `text-{color}` — minimum 4.5:1 contrast ratio against frosted table surface |
