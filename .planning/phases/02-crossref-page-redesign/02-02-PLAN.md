## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Component → DOM | PillChip renders user-visible content (file names, status text) |
| Upload helper → API | XHR upload sends file data to backend endpoint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Spoofing | PillChip status display | accept | Status is data-driven from API response, not user-controlled. The component only renders, never modifies status. Displaying via a `<span>` without interaction handlers prevents click-jacking. |
| T-02-05 | Tampering | uploadWithProgress XHR | mitigate | The function only uploads files to the backend API. The backend's `validate_upload()` function (already existing) validates file type and content server-side. XHR is used for progress reporting only — no additional attack surface. |
| T-02-06 | Information Disclosure | uploadWithProgress error messages | accept | Error responses from backend may contain details about the file (e.g., "invalid format"). This is the same surface as the existing `fetch`-based `api.crossref.upload()`. |
</threat_model>
