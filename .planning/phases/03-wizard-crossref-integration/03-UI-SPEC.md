## Spacing Scale

Declared values (multiples of 4 — Apple 8px base unit with sub-base adjustments):

| Token | Value | Usage |
|-------|-------|-------|
| xxs | 4px | Icon gaps, inline padding, status dot spacing |
| xs | 8px | Table cell vertical padding, compact element gaps |
| sm | 12px | Button padding vertical (8–12px), chip internal spacing |
| md | 17px | Default horizontal padding (Apple body line-height constant) |
| lg | 24px | ConfiguratorCard internal padding, section inner spacing |
| xl | 32px | Layout gaps between cards, wizard step content margins |
| xxl | 48px | Major section breaks, wizard section separation |

**Exceptions:**
- **Match key chips (column mapping):** 8px vertical × 16px horizontal (compact chip layout for multiple keys)

- **Icon-only buttons (add/remove match key):** 36px × 36px (nested within card, not primary navigation; 44px minimum applies to primary CTAs only)

## Interaction Contracts

### 1. Crossref Step (subStep 0 → 1 → mapping)

```
flowchart
  subStep=0: [Yes/No choice]
    Yes → set enableCrossref=true, subStep=1 (upload)
    No  → skip crossref, move to template step
  
  subStep=1: [Upload existing file OR upload new file]
    Upload new → api.crossref.upload() → refresh file list → select file → load columns
    Select existing → api.crossref.get(id) → load columns → proceed to mapping
    
  mapping: [Configure match keys + output columns]
    - Auto-suggest match key(s): compare crossref column names with extraction template column names
      (case-insensitive, trimmed). If name match found → pre-select as match key
    - User can add more match keys (multiple compound keys) via "Agregar otra clave"
    - User can remove match keys via "Quitar" link
    - Output columns: show crossref columns as toggleable chips
      - Smart default: suggest columns NOT in extraction template (avoids duplication)
    - "Confirmar correspondencia" button → store mapping config → move to template step
```

### 2. Match Preview in Review Step

```
flowchart
  User enters review step (after extraction completes)
    ↓
  Check: enableCrossref && selectedCrossrefId && mapping configured?
    ├── No → render existing review step (no crossref preview)
    └── Yes → fetch match results
        ↓
  Display summary bar: "X registros coinciden • Y registros sin coincidencia"
    With green/matched and amber/unmatched pill-style counters
    ↓
  Two expandable sections:
    ├── "Concordancia" (green tinted header):
    │   Table: extraction columns (left) | crossref columns (right)
    │   Each row = one matched record
    │
    └── "Sin concordancia" (amber tinted header):
        Table: extraction columns only, crossref columns shown as blank/gray dashes
        Each row = one unmatched record
```

### 3. Auto Re-Match on Mapping Change

```
If user returns to crossref step, changes column mapping, then returns to review step:
  → Automatically re-fetch match results with new mapping
  → Show loading state on match preview during re-fetch
  → No manual "Re-match" button needed
```

### 4. State Machine

| State | UI | Behavior |
|-------|----|----------|
| Crossref disabled | crossref subStep=0 → user selects "No, omitir" | Skip to template, no match data in payload |
| Crossref enabled, no file | subStep=1 with empty file list | Show empty state + upload dropzone |
| Crossref enabled, file selected | subStep=1 with file selected + mapping UI | Load columns, auto-suggest matches |
| Crossref enabled, mapped | subStep=2 or pushed to template step | Mapping config stored in state |
| Extraction complete with crossref | Review step shows match preview | Summary + expandable sections |
| Mapping changed, re-entering review | Review step match preview | Auto re-fetch, show loading overlay on preview |
| No crossref configured on extraction | Review step (no change) | Existing review step behavior |
