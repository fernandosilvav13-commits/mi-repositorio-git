# Plan 12-02: Five Concrete Rule Implementations

**Phase:** 12 - Post-Processing Rules Expansion
**Plan:** 02
**Wave:** 1
**Objective:** Implement the five inference rules: nationality, DOB, years of experience, education level, email domain.
**Gap closure:** false

## Tasks

### Task 1: NationalityRule

File: `backend/app/services/rules/nationality_rule.py`

- Pattern-match nationality indicators in text
- Maps to field: NACIONALIDAD
- Patterns: "nacionalidad: Chilena", "Chileno/a", "Peruano/a", etc.
- Also checks for explicit mentions like "Nacionalidad: XX"

### Task 2: DateOfBirthRule

File: `backend/app/services/rules/dob_rule.py`

- Pattern-match dates near "FECHA NACIMIENTO", "NACIMIENTO", "Fecha de Nacimiento"
- Maps to field: FECHA_NACIMIENTO
- Supports DD/MM/YYYY, DD-MM-YYYY, DD de Mes de YYYY formats

### Task 3: YearsExperienceRule

File: `backend/app/services/rules/experience_rule.py`

- Analyzes date ranges in work experience section
- Calculates total years of professional experience
- Maps to field: ANIOS_EXPERIENCIA
- Sums differences between start/end dates

### Task 4: EducationLevelRule

File: `backend/app/services/rules/education_rule.py`

- Detects highest education level from text
- Maps to field: NIVEL_EDUCACION
- Levels: "TECNICO", "PREGRADO", "POSTGRADO", "MAGISTER", "DOCTORADO"
- Keyword-based detection

### Task 5: EmailDomainRule

File: `backend/app/services/rules/email_rule.py`

- Extracts email domain type from TELEFONO/EMAIL field
- Maps to field: DOMINIO_EMAIL
- Detects: "gmail.com", "outlook.com", "correo.cl", "empresa.cl", "universidad.cl"

### Task 6: Rule auto-registration

All rules auto-register with RuleRegistry on import.

### Task 7: Unit tests for all rules

File: `backend/tests/test_rules.py`

- Test each rule with known inputs
- Test edge cases (missing data, edge patterns)
- Test registry registration

## Verification

- Each rule produces correct output for known inputs
- Rules register with RuleRegistry automatically
- Edge cases handled gracefully
