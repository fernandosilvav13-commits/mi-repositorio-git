# PLAN — Fix clean_text() Proper Noun Casing

## Goal

Surgical fix to `clean_text()`: remove blanket `.lower()` and add `re.IGNORECASE` so redundant-phrase removal works regardless of casing. Add unit + integration tests.

## Threat Model

```
Trust boundaries:
  ┌─────────────┐     preprocess_cv_text()     ┌──────────────┐
  │  User Input  │ ──────────────────────────►  │  LLM Service  │
  │  (CV text)   │                              │  (Gemini)     │
  └─────────────┘                              └──────────────┘
       Trust: untrusted                              Trust: semi-trusted

STRIDE Register:
  S/T Spoofing/Tampering:  LOW — Input text is untrusted; regex sub with IGNORECASE
                           on user-supplied text could theoretically match attacker-
                           controlled patterns. Mitigated: REDUNDANT_PHRASES are
                           fixed literal strings.  →  ACCEPT
  R   Repudiation:         N/A — no audit requirement
  I   Info Disclosure:     LOW — removing .lower() preserves original text casing,
                           which is less processed than before. No net increase in
                           disclosure risk.  →  ACCEPT
  D   DoS:                 LOW — re.sub with IGNORECASE has negligible performance
                           difference from case-sensitive.  →  ACCEPT
  E   Elevation:           N/A — no auth, no privilege boundary crossed
```

### Task 2 — Create unit + integration tests for preprocessor

**Action:**
- Create `backend/tests/test_preprocessor.py`
- Test file must use the same conventions as existing tests (pytest, no TestClient needed for pure function tests)
- Test scenarios (D-01 through D-08 locked decisions guide these):

  **Unit tests for `clean_text()`:**
  1. All-caps proper noun preservation: `clean_text("MARÍA GARCÍA")` → output must contain `"MARÍA GARCÍA"` (not lowercased) — tests D-01
  2. Redundant phrase removal with IGNORECASE: `clean_text("Hoja de Vida - María García")` → output is `"María García"` — tests D-02
  3. Redundant phrase removal all-caps: `clean_text("CURRICULUM VITAE")` → output is `""` (empty, removed) — tests D-02
  4. Redundant phrase with accent (lowercase input): `clean_text("currículum vitae")` → output is `""` — tests D-03 (diacritics preserved in matching)
  5. Whitespace normalization: `clean_text("  spaced   text  ")` → `"spaced text"` — tests existing behavior unchanged
  6. Empty/blank input: `clean_text("")` → `""`, `clean_text("   ")` → `""`
  7. Mixed: `clean_text("Curriculum Vitae - Juan Pérez LÓPEZ")` → `"Juan Pérez LÓPEZ"` — verifies proper noun casing survives alongside redundant phrase removal

  **Integration tests for `preprocess_cv_text()`:**
   8. Full pipeline preserves casing when sections are found: `preprocess_cv_text("CURRICULUM VITAE\nNOMBRES: MARÍA GARCÍA LÓPEZ\nRUT: 12.345.678-9")` → output contains `#NOMBRES:` with the name in its original casing (e.g., `"MARÍA GARCÍA"`) — tests D-07
   9. Full pipeline fallback (no sections matched): input like `"some random text without any keyword headers"` → returns cleaned raw text preserving original casing (tests fallback path)
   10. Full pipeline with typical CV header: input `"Hoja de Vida\nNOMBRES: Juan Pérez\nRUT: 12.345.678-9"` → redundant phrase removed, name casing preserved, RUT extracted

- Do NOT call external services (LLM, database) — all tests must be pure function tests
- Use `pytest` and `pytest-asyncio` only if testing async functions; `clean_text()` and `preprocess_cv_text()` are synchronous
- Follow existing test conventions from `backend/tests/test_excel_styling.py`
