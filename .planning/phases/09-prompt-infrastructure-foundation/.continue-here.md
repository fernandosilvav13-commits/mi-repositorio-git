# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: PowerShell quoting — All bash commands with nested quotes (double quotes inside single-quoted `-c` args) break on this host because PowerShell strips/modifies quotes before passing to WSL. Always write Python scripts to temp files and execute via `wsl python3 <script>` instead of inline `-c` strings. File writes from opencode tools (Write tool) go to Windows paths; use the `C:\Users\ferna\AppData\Local\Temp\opencode\` directory and reference via `/mnt/c/Users/ferna/AppData/Local/Temp/opencode/` in WSL.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Nested quotes in wsl bash -c | PowerShell strips/mangles nested double quotes inside bash -c strings | blocking | Always write Python logic to a temp .py file, execute with `wsl python3 /mnt/c/.../script.py` |
| Forgetting --break-system-packages | Ubuntu 24.04 system Python blocks pip install without flag | advisory | Always pass `--break-system-packages` to pip install in this environment |
