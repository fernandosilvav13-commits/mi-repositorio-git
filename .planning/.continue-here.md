## Critical Anti-Patterns (do NOT repeat these)

- [ANTI-PATTERN]: WSL UNC path quoting with PowerShell — heredocs and pipeline chars get mangled by PowerShell. Use the Write tool to create `.sh` script files in `/tmp/` instead of inline heredocs. → Write script files to the filesystem first, then execute with `wsl sh -c "path/to/script.sh"`.
- [ANTI-PATTERN]: sudo-requiring system packages — sudo needs interactive password in WSL. Download .deb packages with `apt-get download`, extract with `dpkg-deb -x`, and set LD_LIBRARY_PATH. → Keep all dependencies local to the project directory.
- [ANTI-PATTERN]: `wsl -c` without `sh` — `wsl -c "command"` is not valid PowerShell syntax. Always use `wsl sh -c "command"`.
